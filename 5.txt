							 Язык Web-программирования HTML
Язык HTML - (Hyper Text Markup Language) является основным языком Web-программирования и в переводе озночает 'Язык гипертекстовой разметки' 
впервые данный язык был опубликован в 1993 году разрабом данного языка является WHATWG Концорциум Всемирной паутины, Инжинерный совет с момента 
появления данного языка до сегоднящнего дня были разработаны различные версии языка HTML
1-HTML 1.0 1993
2-HTML 2.0 24.11.1995
3-HTML 3.0 28.03.1995
4-HTML 3.2 14.01.1997
5-HTML 4.0 18.12.1997
6-HTML 4.01 24.12.1999
7-HTML5 28.10.2014
8-HTML 5.1 17.12.2012	1.11.2016
9-HTML 5.2 14.12.2017
10-HTML 5.3 24.12.2018

Основные команды языка HTML называют "тег"-ами существуют 2 вида тегов нечетные и четные. НЕЧЕТНЫЕ ТЕГИ имеют только откытие т.е <имя_тега>. 
ЧЕТНЫЕ ТЕГИ имеют и откытие и закытие закрытие тега от его откытия отличается  использование символа / перед названием тега
<имя_тега>...</имя_тега>. Существуют так же теги которые могут использоваться в обеех видах.
Дополнения используемые вместе с тегами которые обозначают характеристики тега называются АТРИБУТами тега. В HTML тоже существует использование 
коментарий соответсвенно для этого используется тег "<!-- текст_коментария -->" на подобтт коментария существует другой тег коментирующий 
некоторое свойство документа <!DOCTYPE> данный тег используется для того чтобы объявить тип документа язык на котором написан текст документа 
тип описиваемого документа модификатор доступа к документу а так же имя и адрес документа 
"<!DOCTYPE HTML>"	*.html
"<!DOCTYPE PHP>"	*.php

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 //EN"http://www.w3.org/TR/html14/struct.dtd">
Теги связанные с документом 


1)	<html>...</html> - Является самым основным тегом через которого создается документ Web-страницы. Является контейнером внутри которой указывается 
другие теги содержащие приближайщую структуру или данные web-страницы. Основыми атрибутами этого тега является 
title-для добовления спливающей подсказки
manifest-указывает файл манифеста необходимое для создания офлайн-приложения
xmlns-указывает пространство имен для XHTML документов 
Кроме названных атрибутов для данного тега HTML так же доступны 


2)	<head>...</head> - цель которых помочь браузеру в работе с данными т.е внутры тега находятся мето теги которые используются для хранения
информаци предназначеной для браузеров и поисковых систем. <head> является контейнкром используется для создания заголовочной части документа 
Внутри <head> допускается использование тегов <base>, <basefontt>, <bgsound>, <link>, <meta>, <script>, <style>, <title>


3)	<title>...</title> - определяет заголовок документа. Он не является частью документа и не показывается напрямую на web-странице


4)	<meta>...</meta> - определяет метатеги которые используется для хранения информации предназначенной для браузеров и поисковых системы
Во время создания Web-документов разрещается использовать несколько тегов мета для более быстрого поиска и нахождения документов основными
атрибутами тега является cotent-устанавливается значения атрибута заданных другими атрибутами этого же тега. name-имя метатега
http-equiv - предназначен для конвертации метатега в заголовок http, charset устанавливает кодировку символов. Тег мета можно использовать в
след двух видов


5) <link> - устанавливает связ с внещним документов вроде файла с стилями или со шрифтами. В отличии от других тегов работающий со ссылками
этот тег размещается всегда внутри контейнера <head> и не создается ссылку основные атрибуты charset устанавливает кодировку связанного с 
кодировкой, href - устанвливает путь к связаемому файлу media - определяет устройсва для которого следует применять стиловое оформление 
rel - определяет отношение между текущим документом и файлом на который делается ссылка,  type  MIME-тип данных подключаемого файла, size -
укзывает размер иконок для отображения 


6) <style>...</style> - предназначен для ситилей элементов Web-страниицы. Основные атрибуты media - определяет устройство вывода для работы с
которым  предназначенно работа стилей, type - сообщает браузеру какой синтаксис использовать чтобы правильно интерпритировать стили.

7) <script>...</script> - предназначен для описания скриптов может содержать ссылку на программу или её текст на определенном языке. Код скрипта
может быть указан внутри кода страницы или в каком нибудь внешнем файле. Если код скрипта распологается во внещнем файле тогда его можно 
подключить к любому HTML документу такой подход позволяет использовать один и тот же скрипт к нескольким различным HTML документом одновременно.
скрипты могут помещаться или распологаться либо в части заголовки либо в теле страницы :) Если скрипты размещены в заголовке документа то они 
выпольняются в первую очередь. Атрибуты тега: defer откладывает выпольнения скрипта до тех пор пока вся страница не будеть загружена полностью,
language - устанавливает ЯП на котором написан скрипт src - адрес скрипта из внещнего файла для импортов текущего документа, type - определяет 
тип содержимого тега скрипт. Закрытие тега обязательна !!!!


8) <body>...</body> - предназначен для хранение содержания Web-страницы(контента) отображаемого в окне браузера. Основные атрибуты для тега: 
alink - background - задает фоновый рисунок на web-странице bgcolor-задает цвет фона web страницы, bgproperties - определяет прокручивать фон
совместно с текстом или нет, bottommargin - отступ от нижнего края окна браузера до контента, leftmargin - отступ от левого окна браузера до
нового контента, link - устанавливаеи цвеи ссылок на web-страницы, rightmargin - отступ от правого окна браузера до нового контента text -
устанавливает цвет текста в документе topmargin - отступ от верхного окна браузера до нового контента, vlink - цвет посещенных ссылок, так же
для данного тега доступны универсальные атрибуты и события 


9) <header>...</header> - используется для того чтобы создать шапку сайта или раздела в которой обычно распологается заголовок.



10)<footer>...</footer> - предназначен чтобы создать строку состояния сайта или раздела и в нем может быть расположен имя автора разраба дата 
документа и другие контактные информации 



11) Определяет блог с боку от контента для размещения рубрик ссылок на архив такой блок как правило называется side bar или боковая панель 
для данного тега доступны универсальные атрибуты закрывающая часть обязательная
